# AI 每日技术日报 — 设计文档

> 日期：2026-06-27
> 状态：待审核

## 1. 项目概述

面向科技爱好者和普通开发者的 AI 技术日报网站。每日自动聚合 RSS 源，通过 AI 筛选、摘要、翻译，生成 5-10 篇精选文章，支持中英文切换和语音播放。

**目标用户**：科技爱好者 / 普通开发者
**使用场景**：个人 / 小圈子，无需用户系统

---

**核心流程**：
1. Cron Job（每日定时）→ 抓取 RSS → AI 筛选+摘要 → 翻译 → 存储
2. 用户访问 → 读取当日数据 → 渲染页面 → 按需调用 TTS 生成语音

**数据存储**：JSON 文件，每日一个（`data/YYYY-MM-DD.json`），无需数据库。

## 3. 内容管线（Content Pipeline）

### 3.1 RSS 抓取

使用 `rss-parser` 库抓取预配置的 RSS 源。

**初始源列表**：
- TechCrunch
- The Verge
- ArsTechnica
- Hacker News（Top Stories）
- OpenAI Blog
- Google AI Blog
- 36氪
- 少数派

**处理**：设置超时和重试机制，单个源失败不影响整体。

### 3.2 筛选

- **去重**：按 URL 去重，避免重复推送
- **过滤**：AI 判断是否与 AI/科技相关，排除广告和水文
- **排序**：按重要性/相关性打分，取 Top 5-10

### 3.3 AI 摘要

调用 LLM API（OpenAI / DeepSeek），输入原文标题+正文，输出：
- 中文摘要（2-3 句话）
- 英文摘要（2-3 句话）
- 关键词标签（如 #LLM #OpenAI #机器人）

### 3.4 数据结构

每日生成 `data/YYYY-MM-DD.json`：

```json
{
  "date": "2026-06-27",
  "articles": [
    {
      "id": "xxx",
      "title": { "zh": "标题", "en": "Title" },
      "summary": { "zh": "摘要...", "en": "Summary..." },
      "source": "TechCrunch",
      "sourceUrl": "https://...",
      "tags": ["LLM", "OpenAI"],
      "coverImage": "https://..."
    }
  ]
}
```

## 4. 前端页面设计

### 4.1 页面结构

**① 顶部 Header**
- 左侧：日期 + "AI 日报" 标题
- 右侧：中/英切换按钮 + 深色模式切换

**② 文章列表（主体）**
- 卡片式布局，每张卡片包含：
  - 封面图（如有）
  - 标题（中/英，跟随语言切换）
  - 摘要（中/英）
  - 来源标签 + 关键词标签
  - 🔊 语音播放按钮
- 支持「播放全部」按钮 — 连续播报所有文章

**③ 底部 Footer**
- "Powered by AI" 标识
- 数据源列表

### 4.2 设计风格

- 简洁现代，参考 Readhub / 少数派日报风格
- 主色调：浅色背景 + 蓝色强调色
- 卡片圆角、轻阴影，留白充足
- 移动端优先响应式设计

### 4.3 页面路由

- `/` — 当日日报
- `/?date=2026-06-26` — 查看历史日报

## 5. 语音播放（TTS）

### 5.1 播放模式

- **单篇播放**：点击文章卡片上的 🔊 按钮，播放该篇摘要
- **连续播放**：点击「播放全部」按钮，按顺序播报所有文章，文章间有短暂停顿

### 5.2 播放器 UI

- 底部固定的 mini 播放器条（类似播客播放器）
- 显示：当前播放文章标题 + 播放/暂停 + 进度条 + 语言切换
- 播放全部时显示「第 3/8 篇」

### 5.3 TTS 调用策略

- **懒生成**：用户点击播放时才调用小米 TTS API
- **缓存**：生成的音频文件缓存到 `public/audio/YYYY-MM-DD/` 目录
- 按需生成对应语言版本
- 音频格式：MP3

## 6. 中英文切换

### 6.1 切换粒度

- 全局切换，影响所有内容（标题、摘要、UI 文案）
- 用户选择记住在 `localStorage`，下次访问自动应用

### 6.2 内容翻译策略

- **文章标题和摘要**：在内容管线阶段已由 AI 生成双语版本，直接读取
- **UI 文案**：使用简单的字典文件实现

### 6.3 切换交互

- Header 右侧按钮，点击切换，带过渡动画
- 显示当前语言标识：「中」/「EN」

### 6.4 URL 设计

- 不用 URL 路径区分语言（如 `/zh` `/en`），保持 `/` 干净
- 语言状态纯客户端管理

## 7. 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | Next.js 14+ (App Router) | SSR + API Routes + Cron |
| 样式 | Tailwind CSS | 快速开发，移动端适配方便 |
| RSS 抓取 | `rss-parser` | 轻量，支持主流 RSS/Atom |
| AI 摘要 | OpenAI API / DeepSeek | 筛选 + 摘要 + 翻译 |
| TTS | 小米 TTS API | 中英文语音合成 |
| 数据存储 | JSON 文件 (`data/` 目录) | 个人项目足够 |
| 音频缓存 | `public/audio/` 目录 | 静态文件，Vercel 直接托管 |
| 国际化 | 自定义字典文件 | 简单场景 |
| 部署 | Vercel | Cron Jobs + 自动部署 |

**开发依赖**：TypeScript、ESLint + Prettier、`date-fns`

## 9. Cron 配置

`vercel.json` 配置每日早上 8:00（北京时间）触发定时任务：

```json
{
  "crons": [
    {
      "path": "/api/cron",
      "schedule": "0 0 * * *"
    }
  ]
}
```

> 注：Vercel Cron 使用 UTC 时间，UTC 0:00 = 北京时间 8:00。
